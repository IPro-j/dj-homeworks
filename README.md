Сервис транслитерации текста
Веб‑сервис для транслитерации русского текста в латиницу с историей последних запросов.

Функционал
Транслитерация русского текста в латиницу (например, «вода» → «voda»).

Динамическое обновление результата без перезагрузки страницы.

Сохранение истории запросов в LevelDB.

Получение последних N записей истории через API.

Технологии
Backend: Django, Django REST Framework.

Транслитерация: библиотека transliterate.

База данных: LevelDB (через plyvel).

Frontend: HTML, CSS, JavaScript (AJAX‑запросы).

Установка и запуск
1. Установка зависимостей
bash
pip install django djangorestframework transliterate plyvel
2. Клонирование и настройка проекта
bash
git clone <url-репозитория>
cd translit-service
3. Настройка проекта
Создайте и примените миграции:

bash
python manage.py makemigrations
python manage.py migrate
4. Запуск сервера
bash
python manage.py runserver
Сервис будет доступен по адресу: http://127.0.0.1:8000/

API‑эндпоинты
POST /api/
Транслитерирует текст и сохраняет результат в историю.

Запрос:

json
{"data": "текст для транслитерации"}
Ответ (успех):

json
{"status": "success", "data": "transliterated text"}
Ответ (ошибка):

json
{"status": "error", "message": "описание ошибки", "data": null}
GET /history/?n=N
Возвращает последние N транслитерированных текстов.

Параметры:

n — количество записей (по умолчанию 5).

Ответ:

json
{"data": ["voda", "apishka", "privet"]}
Использование
Откройте главную страницу: http://127.0.0.1:8000/.

Введите текст в поле «Русский текст» — транслитерированный вариант появится во втором поле автоматически.

Нажмите кнопку «Загрузить историю», чтобы увидеть последние переводы.

Для работы с API используйте инструменты вроде curl или requests.

Примеры запросов
Транслитерация через curl
bash
curl -X POST http://127.0.0.1:8000/api/ \
  -H "Content-Type: application/json" \
  -d '{"data": "апишка"}'
Получение истории через curl
bash
curl http://127.0.0.1:8000/history/?n=3

Особенности реализации
История запросов сохраняется в LevelDB (папка transliteration_db в корне проекта).

Записи сортируются по убыванию времени создания (новые первыми).

При некорректном параметре n в /history/ возвращается 5 записей.

CSRF‑защита включена для POST‑запросов.

Примечания
База LevelDB создаётся автоматически при первом запросе.

Для очистки истории удалите папку transliteration_db.

Убедитесь, что у процесса есть права на запись в директорию проекта.
